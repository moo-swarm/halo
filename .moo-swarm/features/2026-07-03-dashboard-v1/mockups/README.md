# Mockups — Halo Dashboard v1

Visual prototypes for the build phase. Open each `.html` file in a browser to view.

| File | Shows | Purpose |
|------|-------|---------|
| `desktop-full.html` | Full 3-column layout, light theme | Reference for all sections, desktop proportions |
| `mobile-full.html` | Single-column stack, dark theme | Mobile reference with dark mode |
| `section-states.html` | Loading → Loaded → Empty → Error | Build reference for all four data states |
| `pipeline-states.html` | All passing vs one failing | Two pipeline configurations |
| `filter-interaction.html` | Unfiltered → filtered table | Filter UX for Issues & PRs |

## How to use

Open in any modern browser. These are static HTML/CSS — no build step, no data loading.
They illustrate the visual design, layout, and states for the build team (Cmok 🐉).
